## 2. Déclarations — Création manuelle (formulaire guidé)

### Backend

**Endpoint ajouté :** `POST /declarations`

**DTO `CreateDeclarationDto` :**
```typescript
serviceId: string          // obligatoire
rgaaVersion?: string       // défaut "4.1"
dateAudit?: string         // ISO date
contactName?: string
contactEmail?: string
contactPhone?: string
pages?: CreatePageDto[]    // tableau de pages { name, url, pageType? }
```

Crée la déclaration avec ses pages auditées en une seule requête.

### Frontend (`/declarations/create`)

**Fichier :** `frontend/src/app/features/declarations/create/create-declaration.component.ts`

Formulaire en **3 étapes** via `p-stepper` :

1. **Informations générales** — service, version RGAA, date d'audit, contact
2. **Pages auditées** — ajout/suppression de pages avec nom, URL, type
3. **Récapitulatif** — confirmation avant création

## 3. Déclarations — Page de détail

**Fichier :** `frontend/src/app/features/declarations/declaration-detail.component.ts`

### Résumé en 4 métriques
- Taux de conformité (avec barre de progression et code couleur)
- Nombre total de critères
- Nombre de pages auditées
- Nombre de critères non-conformes

### Onglet Critères
- Tableau paginé (25 lignes par page)
- **Tri** sur toutes les colonnes (référence, intitulé, niveau, statut, impact)
- **Filtres** : recherche texte globale + filtres dropdown sur statut et impact
- Tags colorés : C (vert) / NC (rouge) / NA (gris)

### Onglet Pages
- Liste des pages auditées
- **Compteur de non-conformités par page** (calculé côté frontend depuis les `affectedPages` des critères NC)
- Badge rouge si NC > 0, vert si 0

### Onglet Contact
- Informations de l'auditeur (nom, email, téléphone)

### Dialog d'édition
- Modification du statut (DRAFT / PUBLISHED / ARCHIVED)
- Modification de la version RGAA, date d'audit, informations de contact

## 7. Corrections de bugs

### `Express.Multer.File` — TS2694
Le type `Express.Multer.File` n'était pas résolu dans le contexte Alpine.
**Fix :** import explicite `import type { File as MulterFile } from 'multer'`.

### `ValidationPipe` — 400 Bad Request sur POST /services
La pipe globale (`whitelist: true`, `forbidNonWhitelisted: true`) rejetait les propriétés sans décorateurs.
**Fix :** ajout des décorateurs `class-validator` (`@IsString`, `@IsNotEmpty`, `@IsOptional`) sur `CreateServiceDto` et `CreateDeclarationDto`.

### `FileTypeValidator` — rejet des .xlsx
Le validateur comparait la regex `/\.(xlsx|xls|ods)$/` contre le **MIME type** du fichier, pas l'extension.
**Fix :** regex adaptée aux MIME types réels :
```typescript
fileType: /(spreadsheetml|ms-excel|opendocument\.spreadsheet)/
```

### Stepper Angular — TS2345
`(valueChange)` émettait `number | undefined` incompatible avec `signal.set(number)`.
**Fix :** `activeStep.set($event ?? 1)`
